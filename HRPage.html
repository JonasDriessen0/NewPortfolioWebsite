Hollow Reverie

A Short Linear Horror Game

About the Project

Hollow Reverie is a short linear horror game made for the Goedware Zero UI Game jam. It takes place in an abandoned warehouse of some sort where you have a tablet that displays usefull information and features a sonar and a flash ability to help you along your way. You are also being hunted by an unkown creature that has no sight but has amazing hearing so you need to be carefull and quiet in an attempt to survive. I learned a lot of new unity skills throughout this project but i also learned a lot of game design skills while designing the levels.

The Monster

I thought id get right into what the most challenging part of this project was: The Monster.

The monster has the appearence of a smoke cloud i chose this because i didnt think that i wouldve been able to make a scary fully animated monster in the limited timeframe i before the game jam submissions would end, so i chose this as the player cant really fully figure out what it is and the most prominent fear any human has is the fear of the unkown so i think it works pretty well!

It also definetly helps that the monster makes a terrifying noise when youre near it, see for yourself:

Monster Behavior

To start off, the main way in which the monster behaves is down to either one of these states; Wandering or Hunting.

Lets start off with how the wandering state works.

The wandering state is pretty simple, every 5 seconds (time determined by the wanderTimer variable) it picks a random spot on the navmesh within the wanderRadius to and then goes to it.

Lets get into how the monster switches from wandering to hunting.

As i mentioned before the monster is blind so it reacts purely to sound. To achieve that i first find all audiosource elements in the scene then i determine the loudest and closest sound within range then at last the monster starts heading towards the previously determined audiosource and adjust the monsters speed based on the audiosources volume and distance from the monster.

The Hunting state is activated in the DetectSounds function and when activated the Hunt function makes sure to switch back to wandering when it is finished hunting.
